# DexterousHands — Install Recipe

Minimum commands to reproduce this benchmark container from a fresh clone. Generated by `benchmark-env-generator` skill (sub-skill of `env-generator`).

## Host prerequisites

- Docker (>= 24) and the Compose v2 plugin
- NVIDIA driver + NVIDIA Container Toolkit with a GPU visible to `nvidia-smi`
- IsaacGym source placed at `<repo_parent>/isaacgym/` (sibling to this repo). `docker/setup.sh` can auto-extract it from the community image on first run.

## Build the image

```bash
bash docker/setup.sh
```

`setup.sh` extracts IsaacGym from `uvarc/isaacgym:1.0.preview4-cuda11` if the sibling `isaacgym/` is missing, then runs the four build / up / wait / smoke steps end-to-end.

Base image: `nvidia/cuda:11.8.0-cudnn8-runtime-ubuntu20.04`
Python: `3.8` (managed by `uv`)

The repo has `setup.py` + `requirements.txt` but no `uv.lock`, so the image pre-installs via `uv pip install -r requirements.txt` and explicitly pins `numpy==1.23.0` (required by `setup.py`).

## Bring the container up

```bash
docker compose -f docker/docker-compose.headless.yaml up -d
docker exec -it dexterousnhands_container bash
```

For an X11 / GUI session (swap the compose profile — container name is shared, only one may run):

```bash
xhost +local:docker
docker compose -f docker/docker-compose.x11.yaml up -d
```

## First-run sanity checks (inside the container)

```bash
nvidia-smi
python -c "import torch; print(torch.__version__, torch.cuda.is_available())"
python -c "import bidexhands; print(bidexhands.__file__)"
```

The last line must resolve under `/workspace/DexterousHands/` — that proves the mounted source is what the interpreter imports (editable install succeeded).

## Category-specific smoke run

### RL (IsaacGym): native train.py

The benchmark ships its own `train.py` — use it directly. The generated generic-RL scripts (`scripts/run_demo.py`, `scripts/run_eval.py`, `scripts/train_rl_policy.py`) are intentionally absent for IsaacGym benchmarks because IsaacGym has a custom env-construction protocol (argparse + `gymapi` pre-import ordering) that the generic templates would not match.

```bash
docker exec dexterousnhands_container bash -c \
    "cd /workspace/DexterousHands/bidexhands && \
     python train.py --task ShadowHandOver --algo ppo --num_envs 16 --headless --max_iterations 1"
```

A clean exit (or a benign segfault at gym shutdown — known IsaacGym upstream bug) plus a `Learning iteration 0/1` line in the output is the expected smoke-run outcome.

## Stop / clean up

```bash
docker compose -f docker/docker-compose.headless.yaml down
```
