# DexterousHands — Benchmark Guide

Generated by `benchmark-env-generator` skill. English-only by contract; regenerated on every re-run.

## About

DexterousHands (a.k.a. `bidexhands`) is a benchmark of **bi-manual dexterous manipulation** tasks built on NVIDIA IsaacGym. It ships Shadow Hand and Allegro Hand environments with GPU-parallel physics (thousands of envs in a single process), a shaped reward per task, and reference RL / MARL / Meta-RL / Multi-Task / Offline-RL baselines in the same repository.

| Field | Value |
|-------|-------|
| Simulator | NVIDIA IsaacGym (Preview Release 4, PhysX backend) |
| Benchmark category | **RL** (GPU-parallel vectorised envs with shaped reward) |
| Upstream paper / project | "Bi-DexHands: Towards Human-Level Bimanual Dexterous Manipulation" (PKU-MARL) |
| License | Apache-2.0 (check `LICENSE` at repo root) |
| Primary use case | Large-scale RL training for bi-manual manipulation policies; benchmark baselines for MARL / Meta-RL / Offline-RL research |

> **Note**: This is an IsaacGym-based benchmark. It ships its own `train.py` as the native entry point; the generic `scripts/run_demo.py` / `scripts/run_eval.py` / `scripts/train_rl_policy.py` are intentionally absent.

### Task groups

- **Handoff** (Over / CatchUnderarm / CatchOver2Underarm / TwoCatchUnderarm / CatchAbreast) — coordinated bi-manual object transfer
- **Door** (DoorOpenInward / DoorOpenOutward / DoorCloseInward / DoorCloseOutward) — articulated-object interaction
- **Tool-use** (BottleCap / Kettle / Scissors / SwingCup) — contact-rich manipulation with constrained DoF objects
- **Pick-and-place** (BlockStack / PushBlock / GraspAndPlace / LiftUnderarm) — classical manipulation primitives
- **Single-hand** (ReOrientation / Pen) — single Shadow Hand tasks
- **Allegro variants** (AllegroHandOver / AllegroHandCatchUnderarm) — same task on a different embodiment

## Action / Observation spaces

| Field | Value |
|-------|-------|
| Action dim | `20` (single Shadow Hand) / `40` (two Shadow Hands) — set by `self.cfg["env"]["numActions"]` per task |
| Action semantics | Joint-position targets for each finger DoF (24 DoF per Shadow Hand × sign-of-life coupling); continuous torque limits applied by the PhysX controller |
| Observation type | Proprioception + object state + goal state (`num_obs_dict` in each task class); `ShadowHandPointCloud` additionally exposes a point-cloud variant |
| Control frequency | 60 Hz simulation, typically 20 Hz effective control after `controlFrequencyInv=3` |
| Reward signal | Shaped: distance reward + rotation reward + action penalty + success bonus; see `compute_hand_reward` in each task for the exact weighted sum |

## How to use

The skill produced three receipt files at the repo root — `install.md` (how to build), `history.md` (what the skill did), and this file (what the benchmark *is*). The flow below assumes you have already followed `install.md` and have a running container.

### 1. Enter the container

```bash
docker compose -f docker/docker-compose.headless.yaml up -d
docker exec -it dexterousnhands_container bash
```

### 2. Pick a task

Any identifier from the task table above is a valid value for `--task`. Quickest sanity check:

```bash
# List the canonical task IDs that ship with this benchmark
grep -oE '"ShadowHand[A-Za-z]+"|"AllegroHand[A-Za-z]+"' \
    /workspace/DexterousHands/bidexhands/utils/config.py | sort -u
```

### 3. Train with the benchmark's native entry point

IsaacGym benchmarks have a custom env-construction protocol (argparse + `gymapi` pre-import ordering) and ship their own `train.py`. Use it directly rather than a generic `scripts/train_rl_policy.py`:

```bash
# Must run from inside bidexhands/ (train.py uses os.getcwd() to locate cfg/)
cd /workspace/DexterousHands/bidexhands

# Smoke / sanity — 1 iteration with 16 envs, headless
python train.py --task ShadowHandOver --algo ppo --num_envs 16 --headless --max_iterations 1

# Full training — 4096 envs on a single 4090, headless
python train.py --task ShadowHandOver --algo ppo --num_envs 4096 --headless
```

Supported single-agent RL algorithms (via `bidexhands/algorithms/rl/`): `ppo`, `sac`, `td3`, `trpo`, `ddpg`.

For MARL / Meta-RL / Multi-Task / Offline-RL, use the corresponding entry points:

```bash
python train.py --task ShadowHandOver --algo happo     # MARL: happo/mappo/ippo/hatrpo/maddpg
python train.py --task ShadowHandMetaMT1 --algo maml   # Meta-RL
python train.py --task ShadowHandMetaMT4 --algo mtppo  # Multi-Task: mtppo/mtsac/mttrpo
python train.py --task ShadowHandOver --algo td3_bc    # Offline-RL: bcq/iql/td3_bc/ppo_collect
```

### 4. Evaluate a checkpoint

```bash
cd /workspace/DexterousHands/bidexhands
python train.py --task ShadowHandOver --algo ppo --num_envs 64 --test \
    --model_dir logs/ShadowHandOver/ppo/ppo/model_<iter>.pt --headless
```

The `--test` flag disables training updates; `--model_dir` points at a checkpoint written during a previous training run.

## Troubleshooting

- **Task name not recognised**: identifiers are case-sensitive camelCase (e.g. `ShadowHandOver`, not `shadow_hand_over`). The file names under `bidexhands/tasks/` are snake_case but the `--task` flag takes the CamelCase form from `retrieve_cfg`.
- **`Segmentation fault` at end of training**: known IsaacGym upstream issue at `gym.destroy()`. Training itself completed — check for the final iteration log line before the segfault.
- **`ModuleNotFoundError: isaacgym`**: the sibling `isaacgym/` directory is missing on the host. Re-run `bash docker/setup.sh` or manually place IsaacGym Preview 4 at `<repo_parent>/isaacgym/`.
- **GUI apps fail with `cannot open display`**: you are on the headless compose profile. Switch to `docker-compose.x11.yaml` after `docker compose down`.
- **`cfg/*.yaml not found`**: you launched `train.py` from the wrong directory. It uses `os.getcwd()` to resolve the config path — always `cd bidexhands/` first.
